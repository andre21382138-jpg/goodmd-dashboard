# 2026-05-11 작업 정리

## 오늘의 큰 흐름

발주 워크플로우 1라운드 완성 + SaaS화 첫 신호 + 매장 홈 개선.

```
본사 발주진행 ─ (센터재고 표시 / 중복발주 방지)
       │
       ▼
매장 발주확인 ─ (매장이 상품 추가 가능)
       │
       ▼
본사 발주현황 ─ (확정 단계 신설 / 매장추가 라인 식별)
       │
       ▼
엑셀 다운로드 (status='confirmed'만)
       │
       ▼
매장 입고확인 ─ (store_stock 자동 반영)
```

---

## 1. 본사 확정 단계 + 매장재고 자동 반영 (commit `c62a350`, `cc5bad1`, `80f5f3a`)

발주 status 흐름에 `confirmed`를 추가하고 입고 시 매장재고 자동 반영.

### 변경사항
- 새 status: `sent → requested/rerequested → confirmed → received` (text 컬럼이라 DB 스키마 변경 없음)
- 본사 발주현황: `requested`/`rerequested` 행 펼침 시 hq_qty input + ✓ 확정 버튼 → status='confirmed' + items.hq_qty 일괄 update
- 엑셀 다운로드 필터: `status='confirmed' AND exported_at IS NULL`
- 매장 발주확인 탭 필터: check=`['sent']`, receive=`['confirmed','received']`
- handleReceive에 store_stock 자동 가산 로직 — 판매입력 차감의 거울 패턴
- idempotency 가드: `received_qty != null`이면 store_stock 재가산 skip

## 2. 발주진행 탭에 발주완료 표시 (commit `a23105c`, `b6dc79b`)

- fetchAggregation에 `purchase_orders` + `purchase_order_items` 추가 fetch

- 매칭 라인은 체크박스 자리에 📋, 수량 input disabled, 상품명 옆 `📋 발주 완료` 배지
- 전체선택/해제, 단일 토글, 제출 모두 alreadyOrdered 제외
- status 무관 (sent/requested/rerequested/confirmed/received 전부 "발주됨")

## 7. 매장 발주확인 시 상품 추가 요청 (commit `d79ab08` ~ `dc46351`, 11개 커밋)

가장 큰 작업 — 매장이 본사가 보낸 발주에 누락된 상품을 추가해 요청할 수 있게.

### 데이터 식별 규칙
- `hq_qty=0 AND store_qty>0` → 매장이 추가한 라인
- DB 스키마 변경 없음

### 매장 페이지 (`PurchaseOrderMgrPage`)
- `addedItems` state (펼침 닫으면 초기화)
- 검색 useEffect: 2글자 이상 입력 시 300ms debounce → `products` 테이블 name/code ilike, 상위 10개
  - race condition 방지(`cancelled` 플래그) + error toast
- 발주확인 표 마지막에 `+ 상품 추가` 입력행: 검색 input + 드롭다운 + 수량 + `+ 추가` 버튼
- 드롭다운: 이미 발주에 있는 상품은 회색 + "이미 있음" 표시 (선택 불가)
- 추가된 라인: 노란 배경(`#fffde7`) + `🆕 매장추가` 배지 + 수량 input + ✕ 삭제
- 응답 핸들러 두 곳 통합:
  - `handleRequestAsIs`: items store_qty=hq_qty update + addedItems insert
  - `handleRerequest`: editMap 기반 update + addedItems insert
  - confirm 메시지에 추가 상품 개수 표시

### 본사 페이지 (`PurchaseOrderHQPage`)
- 발주현황 펼침 표 상품명 옆에 `🆕 매장 추가` 배지 표시 (식별 규칙 사용)
- 확정 시 hq_qty>0으로 바꾸면 자연히 배지 사라짐
- 0으로 두면 엑셀 출력에서 자동 제외

### 리뷰에서 잡힌 문제 & 수정
- `newQty` 상태가 `handleRerequest` 지역 변수와 shadowing → `addQty`로 리네임 (`5ce57fd`)
- 검색 effect race condition + 에러 미처리 → cancelled 플래그 + 에러 토스트 (`e72fbf3`)
- ✕ 삭제 버튼 type=button 누락 → 추가 (`b027269`)
- 탭 전환 시 addedItems 미리셋 → 정리 (`8f82091`)
- `+ 추가` 버튼 글자 잘림 → whiteSpace:nowrap + flexShrink:0 + minWidth (`dc46351`)
