# 매장매출 - 상품별 집계 뷰 (Design)

작성일: 2026-05-06
대상 파일: `src/pages/sales/SalesListPage.jsx`
관련 메뉴: 본사 → 매출조회 → 매장 매출 (`sales_list`)

## 1. 배경 / 목적

본사에서 특정 상품(또는 키워드 매칭 상품군)의 기간별·점포별 매출 현황을 빠르게 확인할 수 있어야 함. 현재 매장매출 페이지는 개별 판매내역(raw rows)만 표시하기 때문에 다음과 같은 질문에 즉시 답하기 어려움:

- "지난달 '립스틱'으로 검색되는 상품들이 총 얼마나 팔렸지?"
- "특정 상품이 어느 점포에서 가장 많이 팔리고 있지?"
- "이 상품의 일자별 매출 추이는?"

상품 키워드 검색 → 상품별 집계 → (선택) 점포/날짜 분해 드릴다운 흐름으로 위 질문을 한 화면에서 해결한다.

## 2. 사용자 흐름

1. 본사 계정으로 매출조회 → 매장 매출 진입.
2. 페이지 상단 토글에서 **상품별 집계** 선택.
3. 기간(시작일~종료일), 키워드, 필요 시 점포·브랜드 필터 입력.
4. 키워드와 매칭되는 상품들이 표로 집계 표시 (브랜드 / 상품명 / 판매건수 / 수량 / 매출액).
5. 특정 상품 행의 **[점포별]** 또는 **[날짜별]** 버튼 클릭 → 모달 오픈.
6. 모달 상단 토글로 점포별 ↔ 날짜별 보기 전환.

## 3. UI 사양

### 3.1 보기 모드 토글

매장매출 페이지 상단(필터바 위)에 모드 전환 토글을 둔다.

```
[ 📋 판매내역 ]  [ 📊 상품별 집계 ]
```

- 기본 모드: `판매내역` (기존 동작 그대로).
- 모드 상태는 페이지 내부 state로만 관리 (URL 파라미터·세션 영속화 없음).
- 모드 전환 시 필터값(점포/브랜드/기간/키워드/완전반품 토글)은 유지한다.
- 정렬값은 모드별로 의미가 다르므로 모드 전환 시 각 모드의 기본 정렬로 리셋한다.

### 3.2 상품별 집계 모드 - 필터바

기존 매장매출의 필터바를 그대로 사용:

- 점포 select (전체/개별)
- 브랜드 select (전체/개별)
- 시작일 / 종료일 + 어제 / 당월 / 전월 빠른 버튼
- 키워드 input (상품명·브랜드명 부분일치)
- 완전반품 포함 체크박스
- 초기화 버튼

차이점: 정렬 select의 옵션이 모드에 맞게 바뀐다 (3.3 참조).

### 3.3 상품별 집계 테이블

| 컬럼 | 설명 |
|---|---|
| 브랜드 | `brands.name` |
| 상품명 | `products.name` |
| 판매건수 | 해당 상품의 sales 행 수 (반품 행 제외) |
| 총 수량 | Σ 실효수량 (`quantity - returned_qty`, 0 이상) |
| 총 매출액 | Σ (실효수량 × `price`) |
| 상세 | `[점포별] [날짜별]` 버튼 |

- 정렬 옵션: `매출액↓(기본)` / `수량↓` / `판매건수↓` / `상품명`.
- 표 하단 합계 행: `총 N개 상품 · 합계 건수 · 합계 수량 · 합계 매출액`.
- 빈 결과: "조회된 상품이 없습니다".

### 3.4 드릴다운 모달

상품 행의 `[점포별]` 또는 `[날짜별]` 버튼 클릭 시 모달 오픈. 모달 헤더 영역:

```
어딕트 립스틱 (디올)
2026-04-01 ~ 2026-04-30 · 12건 · 14개 · 980,000원
[ 🏬 점포별 ]  [ 📅 날짜별 ]                    [ 닫기 ]
```

클릭한 버튼에 해당하는 보기를 기본 활성화한다.

**점포별 보기 컬럼**: 점포(백화점) / 지점 / 판매건수 / 수량 / 매출액
- 정렬: 매출액↓ 기본.
- 점포 필터가 걸려 있으면 해당 점포만 표시 (4.2 참조).

**날짜별 보기 컬럼**: 날짜 / 판매건수 / 수량 / 매출액
- 정렬: 날짜 오름차순 기본.
- 판매가 없는 날짜는 행을 만들지 않음 (sparse).

모달은 단일 상품에 대한 분해 뷰이며, 닫으면 메인 집계 화면으로 복귀한다.

## 4. 데이터 처리

### 4.1 데이터 소스

- 테이블: `sales`
- 조인: `brand:brands(name)`, `product:products(name)`
- 기존 매장매출 페이지의 select·필터 패턴을 동일하게 사용.

### 4.2 필터 적용 규칙

기존 페이지와 동일하게 서버사이드 필터링 후, 키워드/완전반품 필터는 클라이언트 사이드로 처리한다.

- `fStore` 적용 시: 메인 집계 및 드릴다운 모두 해당 점포 데이터만 사용.
- `fBrand` 적용 시: 해당 브랜드 상품만 집계.
- `fFrom` / `fTo`: `sold_at` 범위 필터.
- `fKeyword`: 클라이언트 사이드에서 `product.name` 또는 `brand.name` 부분일치(소문자 비교).
- `showReturned=false` (기본): 완전반품(`returned_qty >= quantity`) 행 제외.
- 부분반품: 실효수량/매출 계산에 자동 반영(차감).

### 4.3 집계 로직

쿼리 결과 배열을 다음 키로 그룹핑:

- 메인 집계: `product_id` (또는 product_id가 null인 경우 `product.name`)
  - 각 그룹: 행 수 = 판매건수, Σ 실효수량 = 수량, Σ 실효금액 = 매출액
  - 브랜드명·상품명은 그룹 내 첫 행에서 차용
- 드릴다운 점포별: `(store_name, branch_name)` 복합키
- 드릴다운 날짜별: `sold_at` (날짜 문자열 그대로)

기존 페이지 `limit(500)`을 따르되, 집계 모드에서 결과가 잘릴 위험이 있다 (5.1 위험 참조).

### 4.4 반품 처리 헬퍼

기존 함수 재사용:

- `effQty(s) = max(0, s.quantity - s.returned_qty)`
- `effAmt(s) = effQty(s) * s.price`
- `isFullyReturned(s) = s.returned_qty >= s.quantity`

판매건수 카운트 시: `showReturned=false`인 경우 완전반품 행은 카운트에서 제외, 부분반품은 1건으로 카운트한다.

## 5. 구현 메모

### 5.1 위험 / 제약

- **500건 limit**: 기간을 길게 잡고 키워드 필터가 약하면 500건에서 잘려 집계가 부정확해질 수 있음. 이번 스펙에선 기존 limit를 유지하되, fetch 결과 행 수가 정확히 500이면 잘림으로 간주하여 상품별 집계 모드의 합계 행 옆에 경고 배지를 표시한다 ("⚠️ 결과 500건 초과 - 기간을 좁혀주세요"). 판매내역 모드의 동작은 변경하지 않는다.
- **product_id null 케이스**: 과거 데이터에 `product_id`가 비어 있을 수 있음 → 그룹핑 키로 fallback에 `product.name` 사용.
- **모달 외부 클릭/ESC 닫기**: 기본 동작으로 닫히도록 한다.

### 5.2 변경 범위

- 수정 파일: `src/pages/sales/SalesListPage.jsx` 단일 파일.
- 새 파일 생성하지 않음.
- 기존 `판매내역` 보기와 모드 토글로 공존 (회귀 없음).

### 5.3 스타일

- 토글 버튼은 기존 `quickBtnStyle` 패턴(active state 강조)을 재사용.
- 모달은 프로젝트의 기존 모달 스타일이 있으면 따르고, 없으면 fixed overlay + card로 간단 구성.
- 컬럼 정렬: 숫자 컬럼은 `className="r"` (우측 정렬, 기존 클래스).

## 6. 비목표 (Out of Scope)

- CSV 내보내기.
- 점포×날짜 매트릭스(피벗) 뷰.
- 차트/그래프 시각화.
- 필터·정렬 상태의 URL/세션 영속화.
- 특판매출(`biz_sales`) / 강좌매출(`lecture_sales`)에 동일 기능 적용 - 본 스펙은 매장매출(`sales`)에 한정.
